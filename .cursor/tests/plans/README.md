# 测试计划目录

本目录存放各底层框架的**测试计划**（Markdown）。

## 命名与内容

- 建议命名：`{框架简称}-测试计划.md`，例如：
  - `persistence-测试计划.md`（持久化）
  - `executor-测试计划.md`（线程模型）
  - `redis-cache-测试计划.md`（Redis 缓存）
  - `redis-route-测试计划.md`（Redis Route）
- 或使用总览：`底层框架测试计划.md`，内部分章节描述各框架的测试范围与要点。

## 计划应包含

- 目标模块与核心类
- 单元测试要点（可 Mock、无外部服务）
- 集成测试要点（需 Redis/MySQL 时）
- 建议用例列表或检查项

当用户要求对某一框架进行测试时，框架测试 Skill 会优先读取本目录下对应计划；若不存在则先创建计划再编写/执行测试。
