# 底层框架测试计划（总览）

本文档描述四类底层框架的测试范围与要点。针对单一框架执行测试时，可只关注对应章节；若无单独计划文件，以本文为基准。

---

## 1. 持久化框架（slg-support）

**模块**：`slg-support`  
**核心类**：`AsyncPersistenceService`、`BaseRepository`/`BaseMysqlRepository`、`EntityCache`、`PersistenceRetryWrapper`、`PersistenceExceptionUtil`

### 单元测试

- **PersistenceRetryWrapper**：正常一次成功；可重试异常重试至成功/达上限；不可重试异常不重试。
- **PersistenceExceptionUtil**：`isRetryable()` 对各类异常返回符合预期。
- **AsyncPersistenceService**（Mock Repository + Executor）：entity/id 为 null 不调用 repository；有 id 时提交任务并调用 save/insert；findById 的 CompletableFuture 在 executor 执行后完成。
- **EntityCache**（Mock AsyncPersistenceService）：内存 get/put/remove；未命中时调用 findById；过期驱逐时若开启 autoSaveOnExpire 触发 save。

### 集成测试（可选）

- Testcontainers MySQL + BaseMysqlRepository：insert/save/findById/updateField/delete 与数据一致性。
- EntityCache + 真实 AsyncPersistenceService + Repository 的读穿写回场景。

## 2. 线程模型（slg-common）

**模块**：`slg-common`  
**核心类**：`KeyedVirtualExecutor`、`GlobalScheduler`、`Executor`、`TaskModule`、`MultiExecutor`/`SingleExecutor`

### 单元测试

- **TaskModule**：`toKey()`/`toKey(long)` 单链返回缓存 Key、多链按 id 生成；`isMultiChain()` 与枚举一致。
- **KeyedVirtualExecutor**：同 key 串行、不同 key 并发；单链多任务串行；`inThread(module, id)` 在对应线程内为 true；任务抛异常不导致后续任务不执行。
- **GlobalScheduler**：schedule(delay) 约 delay 后执行一次；scheduleWithFixedDelay 间隔 ≥ delay。
- **Executor**：各静态字段非 null，多链传 key、单链不传 key 的用法可编译/运行不抛异常。

### 集成测试（可选）

- 多线程提交大量不同 key 任务，验证完成顺序与并发度、无死锁无漏执行。

## 3. Redis 缓存（slg-redis）

**模块**：`slg-redis`  
**核心类**：`RedisCacheService`、`CacheAccessor`、`CacheModule`、`CacheEntityMeta`/`CacheFieldMeta`

### 单元测试

- **CacheModule**：`buildKey(entityId)` 格式为 `cache:{keyPrefix}:{entityId}`；`fromId(id)` 与枚举一致。
- **CacheEntityMeta/CacheFieldMeta**：从 @CacheEntity 解析的 key 前缀、字段列表、类型与编码正确。
- **CacheAccessor**（Mock StringRedisTemplate）：setField/getField 调用 hash put/get 且 key/field/value 编码正确；getFields/setFields 批量语义一致。
- **RedisCacheService**（Mock RedisTemplate）：set/get/delete 及带过期 set、get 转类型逻辑。

### 集成测试（可选）

- Testcontainers Redis：RedisCacheService、CacheAccessor 真实 set/get/delete/过期与序列化。

---

## 4. Redis Route（slg-net）

**模块**：`slg-net`  
**核心类**：`RedisRoute`、`RedisRoutePublisher`、`RpcRedisRouteConsumerRunner`、`RpcRedisFacade`、`RpcRouteRedisProperties`

### 单元测试

- **RedisRoute**：`getRouteParams()` 返回 `[int.class]`；参数不足时 checkRouteParams 抛异常。
- **RedisRoutePublisher**（Mock RedisTemplate + IRpcRouteSupportService）：publish 使用 getRedisRouteChannel 且 XADD key/body 正确；publishResp 使用 `rpc:route:resp:{callerServerId}`。
- **RpcRedisRouteConsumerRunner**：启动/停止 running 状态正确；可 Mock 消费逻辑验证解码与回调。
- **RpcRouteRedisProperties**：yml 绑定 `rpc.route.redis.*` 默认值与类型。

### 集成测试（可选）

- Testcontainers Redis：Publisher + Consumer，发送一条 RPC 请求，验证 Stream 有记录且消费者能解码/ACK。

---

## 执行策略

- **单元测试**：在各模块 `src/test/java` 下，JUnit 5 + Mockito，不依赖 Redis/MySQL。
- **集成测试**：可选独立模块 `slg-framework-test` 或各模块 `@ActiveProfile("test")` + Testcontainers。
- 运行：`mvn test -pl slg-common` 等按模块执行；集成模块单独 `mvn test -pl slg-framework-test`。
