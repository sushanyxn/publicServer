# 业务测试与造号

## 目标

- 使用 MCP 工具对游戏内具体功能进行实机测试（登录→操作→验证）。
- 帮助用户创建定制账号（造号），通过 GM 指令设置账号状态。
- 当 MCP 不可用时，回退到单元测试或向用户询问。

## 前置知识

执行前必须阅读以下文件：

1. **`.cursor/MEMORY.md`** — 项目架构、模块职责、命名约定
2. **`.cursor/gm/`** — GM 指令文档（速查可用 GM 指令及参数）
3. **`.cursor/protocol/`** — 协议文档（速查模块协议及用途）

## 测试方式优先级

```
MCP 工具（实机测试） > 单元测试 > 询问用户
```

1. **优先 MCP 工具**：除非用户特别要求其他方式，默认使用 MCP 工具进行测试。
2. **MCP 不可用时**：检查对应模块是否有单元测试（`src/test/java` 下），有则执行。
3. **都没有时**：向用户询问希望使用哪种测试方式（编写单元测试 / 手动验证 / 其他）。

## MCP 工具使用

### 可用工具速查

| 工具 | 用途 | 必需参数 |
|------|------|----------|
| `login` | 登录账号 | account, playerId(可选) |
| `list_accounts` | 查看已登录账号 | 无 |
| `get_account` | 查看账号详情 | account |
| `disconnect` | 断开连接 | account |
| `list_heroes` | 查看英雄列表 | account |
| `hero_levelup` | 英雄升级 | account, heroId |
| `gm_command` | 发送 GM 指令 | account, command |

> **注意**：MCP 工具列表会随 slg-client 的开发持续扩展。执行前先确认 MCP Server `project-0-slgserver-slg-client` 已连接且工具可用。若工具不存在或 MCP 未连接，回退到其他测试方式。

### GM 指令速查

测试前如需了解可用 GM 指令，先查阅 `.cursor/gm/` 目录下的 GM 文档。

## 业务测试流程

### 1. 确认测试目标

- 明确用户要测试的功能模块（如英雄、任务、场景等）。
- 确认需要的前置条件（如需要特定等级、特定英雄等）。

### 2. 准备测试账号

- 查看是否有已登录账号（`list_accounts`）。
- 若没有，登录一个测试账号（`login`）。
- 若用户提供了自己的账号名，使用用户指定的账号。
- 若需要新建，使用一个新的账号名登录（首次登录自动创建角色）。

### 3. 设置前置条件

- 通过 GM 指令设置测试所需的前置状态。
- 常见操作：获取英雄（`gainHero`/`gainAllHero`）、设置等级（`setHeroLevel`）等。
- 具体 GM 指令参见 `.cursor/gm/` 文档。

### 4. 执行测试操作

- 使用对应的 MCP 工具执行业务操作。
- 对操作前后的数据进行对比验证。
- 记录每步操作的输入和输出。

### 5. 验证结果

- 通过查询类 MCP 工具确认操作结果（如 `list_heroes` 验证英雄数据变化）。
- 对比预期结果与实际结果。
- 如有异常，记录错误信息。

### 6. 清理与总结

- 测试完成后断开连接（`disconnect`），除非用户要求保持。
- 输出测试总结表格：测试步骤、操作、预期结果、实际结果、是否通过。

## 造号流程

当用户要求"创建一个定制账号"或"帮我造一个号"时：

### 1. 确认需求

- 账号名：用户指定或自动生成（如 `test_hero_1`）。
- 定制要求：用户描述的账号状态（如"满级全英雄"、"3个10级英雄"等）。

### 2. 登录账号

```
login(account=账号名)
```

### 3. 执行 GM 指令定制

根据用户需求，组合执行 GM 指令。例如：
- "全英雄满级" → `gm_command(gainAllHero)` + `gm_command(heroMaxLevel)`
- "拥有1001号英雄且5级" → `gm_command(gainHero 1001)` + `gm_command(setHeroLevel 1001 5)`

### 4. 验证与交付

- 通过查询工具验证账号状态符合要求。
- 向用户报告账号信息（账号名、playerId、当前状态）。

### 5. 缺少 MCP 工具

若造号过程中需要的 MCP 工具不存在（如缺少某模块的查询或操作工具），向用户说明缺失的工具，并询问：
- 是否通过 GM 指令替代
- 是否需要先开发对应的 MCP 工具

## 注意事项

- 测试过程中如果 MCP Server 未连接（工具调用报错），立即告知用户并建议启动 slg-client。
- GM 指令通过 `gm_command` MCP 工具发送，格式为空格分隔的字符串（如 `"gainHero 1001"`）。
- 每次测试操作后都应验证结果，不要假设操作一定成功。
- 造号时如果用户没有指定账号名，使用有意义的命名（如 `test_{模块}_{序号}`）。
