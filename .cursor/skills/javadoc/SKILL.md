# 为包或类添加 JavaDoc 注释

## 输入

- **包**：包路径（如 `com.slg.game.develop.hero`），处理该包及**所有递归子包**下的类。
- **类**：单个类名或全限定类名，仅处理该类。

## 规范（遵循 .cursorrules）

- **注释语言**：所有 JavaDoc 与代码注释使用**简体中文**。
- **类注释**（每个类必须包含）：
  - JavaDoc 格式，内容包含：
    - 类的简要描述（中文，可多行）
    - `@author` 作者名称
    - `@date` 创建日期，格式 `yyyy-MM-dd`
  - 若类已有部分类注释，补全缺失项（如缺 @author、@date），不重复或冲突。
- **方法注释**：
  - 公共方法、受保护方法及对理解逻辑有帮助的私有方法，补充方法说明（一句话概括用途）。
  - 有参数时补充 `@param 参数名 说明`。
  - 有返回值且非 void 时补充 `@return 说明`。
  - 会抛出受检异常或重要运行时异常时补充 `@throws 异常类 说明`。
- **不写废话**：不写“获取 xxx”“设置 xxx”等与签名完全重复的注释；复杂业务或非自解释逻辑可稍详说明。
- **注释与代码一致**：已有注释若与代码逻辑不符（如参数/返回值/异常描述错误、行为描述与实现矛盾），**以代码为准修改注释**，不修改代码逻辑。

## 执行步骤

### 1. 确定范围

- 根据输入解析目标包（递归子包）或唯一类，列出待处理的类。

### 2. 检查已有注释与代码一致性

- 对每个**已有 JavaDoc** 的类、方法：
  - **类注释**：描述是否与类实际职责、成员、用途一致；若已过时或与代码不符，按当前代码修正描述。
  - **方法注释**：对照方法签名与实现检查：
    - @param 数量、参数名是否与签名一致；说明是否与参数在代码中的用途一致。
    - @return 是否与真实返回值、null/异常情况一致。
    - @throws 是否与代码中实际抛出的异常一致（多写或少写都要修正）。
    - 方法说明是否与真实行为一致（如写“不修改 xxx”但实现会修改，则改注释）。
  - **不一致时**：以**代码逻辑为准**，修改注释内容，不修改代码。

### 3. 类注释

- 对每个类：若无类注释则添加完整类注释；若有则补全缺失的 @author、@date 或描述；若已在步骤 2 中发现描述与代码不符则已修正。描述根据类名与职责用中文简要说明。

### 4. 方法注释

- 对每个方法（优先 public、protected，必要时重要 private）：
  - 若无方法 JavaDoc，则添加方法说明及 @param、@return、@throws（按需），且描述需与实现一致。
  - 若已有 JavaDoc，补全缺失的 @param/@return/@throws；若在步骤 2 中已发现与代码不符则已修正；否则不改变原有合理描述。
- 构造函数、getter/setter 若含义明确可简短一行描述；重载方法需区分说明差异。

### 5. 其它

- 字段：若为公共或包级可见、或含义不直观，可加简短行内或 JavaDoc 注释；已有注释若与字段用途不符则修正。
- 不修改任何业务逻辑与代码结构，仅新增、补全或修正注释。

### 6. 收尾

- 快速过一遍：无错别字；@param 名称与源码一致；所有注释与当前代码逻辑一致。

## 简要检查清单

- [ ] 输入已明确为包（递归子包）或唯一类
- [ ] 已检查已有注释与代码逻辑一致性，不符处已按代码修正注释
- [ ] 每个类均有类注释（描述 + @author + @date），且描述与类职责一致
- [ ] 公共/重要方法均有方法注释及必要的 @param、@return、@throws，且与实现一致
- [ ] 注释均为简体中文，无冗余废话
- [ ] 未改动任何业务逻辑与代码结构
