# ADR-004: 协议号按模块分段分配

## 状态

已采纳

## 背景

项目使用 `message.yml` 统一注册所有协议类，每个协议类对应一个全局唯一的协议号。随着模块增多，需要一套清晰的协议号分配策略，避免冲突并便于管理。

## 决策

按以下规则分配协议号：

| 范围 | 用途 |
|------|------|
| 0 | null（系统保留） |
| 1-50 | 基础类型（系统保留） |
| 100-999 | 内部消息（innermessage），每模块预留 10-20 个 |
| 1000+ | 客户端消息（clientmessage），每模块预留约 100 个 |

在 `message.yml` 中按模块分组，注释标明协议号范围（如 `# 登录模块 (1000-1099)`）。

## 理由

1. 分段预留使各模块可独立分配协议号而不冲突
2. 内部消息（100-999）与客户端消息（1000+）有明确界限
3. 按模块分组便于查找、审阅与自动化检查

## 后果

- 新增模块时需在 message.yml 中预留一段协议号范围并注释
- 若某模块协议超出预留范围，需调整分配或使用非连续号段
- protocol-facade-check Skill 可自动检查注册完整性
