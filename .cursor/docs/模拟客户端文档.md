# slg-client 模拟客户端文档

## 1. 模块定位

slg-client 是一个基于 JavaFX + Spring Boot 的游戏客户端模拟器，用于：

- **功能验证**：完全模拟客户端与服务器交互，验证业务功能正确性
- **AI 自动化测试**：通过 MCP 协议（端口 8099，SSE 传输）暴露操作工具，供 Cursor AI 直接调用
- **多账号调试**：单端同时登录多个账号，账号间互不影响
- **协议同步扩展**：跟随服务端新增协议自动扩展对接

不使用任何美术资源，完全使用 JavaFX UI 框架展示内容。

## 2. 技术栈

| 项目 | 选型 |
|------|------|
| UI 框架 | JavaFX 21（OpenJFX） |
| 应用框架 | Spring Boot 3.5.12 |
| 网络层 | 复用 slg-net WebSocket 客户端 + 自定义二进制编解码 |
| AI 接口 | Spring AI MCP Server 1.1.3（spring-ai-starter-mcp-server-webmvc，SSE 传输） |
| 构建 | Maven，父模块为 slgserver |

### 为什么选择 JavaFX

1. **协议零成本复用**：与服务端同 JVM，直接使用 slg-net 中所有 `CM_`/`SM_` 协议类、`MessageCodec`、`WebSocketClientManager`
2. **slg-robot 模式可直接沿用**：已验证的 Java 客户端连接模式（connect -> NetSession -> MessageHandler）
3. **SLG 大场景**：Canvas 支持网格绘制、平移缩放、点击选中，无美术资源时性能充裕
4. **多账号隔离**：每个账号独立 WebSocket 连接 + 独立数据上下文，TabPane 提供 UI 隔离
5. **单语言单进程**：避免 Java + JS 双技术栈的维护成本

## 3. 模块层级

slg-client 属于 **三级业务包**，与 slg-robot 同级：

```
一级: slg-common
二级: slg-net, slg-redis, slg-support
三级: slg-client (依赖 slg-net, slg-support)
```

## 5. 核心架构

### 5.1 启动流程

```
ClientMain.main()
  → Application.launch(ClientApp.class)
    → ClientApp.init()     -- Spring Boot 启动，初始化所有 Bean
    → ClientApp.start()    -- 获取 MainWindow Bean，显示 JavaFX 窗口
    → ClientApp.stop()     -- 关闭 Spring 容器 + JavaFX
```

### 5.2 多账号模型

每个账号拥有独立的上下文，互不影响：

- **ClientAccount**：持有 `NetSession`（独立 WebSocket 连接）、`playerId`、`loggedIn` 状态、`moduleDataMap`（各模块数据）、JavaFX 可观察属性（用于 UI 绑定）
- **AccountManager**：`sessionAccountMap`（session -> account 映射，消息路由用）、`accountMap`（账号名 -> account）、`activeAccount`（当前 UI 聚焦账号）

### 5.3 消息处理流程

```
服务器 SM_ 响应
  → Netty Pipeline 解码
  → ClientMessageHandler.onMessage()
    → NetSession -> AccountManager.getBySession() -> ClientAccount
    → MessageHandlerManager.getHandler() -> MessageHandlerMeta
    → Executor.Client.execute(playerId, handler)  // 按账号串行
      → xxxClientHandler.method(session, message, account)
        → 更新 ClientAccount 数据
        → Platform.runLater() 刷新 UI
```

- **Executor.Client** 是多链执行器（按 playerId 分链），保证同一账号的消息串行处理
- ClientHandler 中的第三参数类型为 `ClientAccount`，框架通过 `isNeedOwner` 自动识别

### 5.4 模块化面板系统

所有业务模块实现 `IClientModule` 接口：

```java
public interface IClientModule {
    String moduleName();           // 导航菜单显示名
    default int order() { return 100; }  // 排序权重
    Pane createPanel(ClientAccount account);  // 创建模块 UI
}
```

- `ClientModuleManager` 在启动时自动扫描所有 `IClientModule` 实现
- 主窗口左侧导航列表自动填充所有已注册模块
- 切换导航项时调用 `createPanel()` 创建对应面板

### 5.5 SLG 大地图

`MapCanvas` 基于 JavaFX Canvas 实现：

- **网格绘制**：按 `CELL_SIZE`（40px）绘制网格线和坐标标注
- **平移**：鼠标拖拽移动视口（cameraX/cameraY）
- **缩放**：滚轮缩放（0.2x ~ 3.0x），以鼠标位置为中心
- **点选**：点击时反算世界坐标，查找对应 cell 上的实体
- **实体渲染**：`MapEntity` 按类型（城池/部队/资源/NPC）用不同颜色的矩形 + 文字标签显示
- **HUD**：左下角显示当前坐标、缩放比例、实体数量

### 5.6 MCP 工具

slg-client 通过 Spring AI MCP Server（SSE 传输）在端口 8099 上暴露以下工具，供 Cursor AI 通过 `CallMcpTool` 直接调用：

| 工具名 | 参数 | 说明 |
|--------|------|------|
| `login` | `account`, `playerId?` | 登录指定账号 |
| `list_accounts` | 无 | 获取所有在线账号 |
| `get_account` | `account` | 获取指定账号信息 |
| `disconnect` | `account` | 断开指定账号 |
| `list_heroes` | `account` | 获取指定账号英雄列表 |
| `hero_levelup` | `account`, `heroId` | 请求英雄升级 |
| `gm_command` | `account`, `command` | 发送 GM 指令 |

工具定义在 `com.slg.client.mcp.ClientMcpTools` 中，使用 `@Tool` 注解（`org.springframework.ai.tool.annotation.Tool`）声明，Spring AI MCP 框架自动生成 JSON Schema 供 AI 发现和调用。

## 6. 命名约定

| 类别 | 命名规则 | 示例 |
|------|---------|------|
| 协议处理类 | `模块名ClientHandler` | `LoginClientHandler`、`HeroClientHandler` |
| 模块面板类 | `模块名Panel`（实现 `IClientModule`） | `PlayerInfoPanel`、`MapPanel`、`HeroPanel` |
| 协议处理包 | `com.slg.client.message.模块名` | `message.login`、`message.hero` |
| UI 面板包 | `com.slg.client.ui.panel` | — |
| 场景组件包 | `com.slg.client.ui.scene` | — |
| MCP 工具包 | `com.slg.client.mcp` | — |

与服务端的命名对比：

| 位置 | 服务端 | 客户端模拟器 |
|------|--------|-------------|
| 协议处理类 | `xxxFacade`（如 `LoginFacade`） | `xxxClientHandler`（如 `LoginClientHandler`） |
| 协议处理注解 | `@MessageHandler` | `@MessageHandler`（相同） |
| 方法签名（已登录） | `(NetSession, 协议类, Player)` | `(NetSession, 协议类, ClientAccount)` |
| 执行器 | `Executor.Player` | `Executor.Client` |

## 7. 开发约束

### 7.1 通用规范

- 遵循项目 MEMORY.md 中的所有编码规范（Lombok、工厂方法命名、日志规范等）
- 所有类必须有 JavaDoc 类注释（简体中文）
- 使用 `LoggerUtil` 进行日志输出，禁止直接使用 SLF4J Logger

### 7.2 协议处理

- ClientHandler 类必须标注 `@Component`，处理方法标注 `@MessageHandler`
- 三参形式 `(NetSession session, SM_Xxx message, ClientAccount account)` 用于已登录协议
- 两参形式 `(NetSession session, SM_Xxx message)` 仅用于登录前阶段
- 处理方法中更新 `ClientAccount` 数据后，通过 `Platform.runLater()` 刷新 UI

### 7.3 UI 开发

- **不使用任何美术资源**，完全使用 JavaFX 原生控件和 CSS 样式
- UI 更新必须在 JavaFX Application Thread 中执行（使用 `Platform.runLater()`）
- 模块面板通过实现 `IClientModule` 接口自动注册，无需修改框架代码
- 面板的 `createPanel()` 接收 `ClientAccount` 参数，用于数据绑定和消息发送
- 使用 JavaFX 可观察属性（`SimpleXxxProperty`）实现数据到 UI 的自动绑定
- **面板绑定模块数据时，必须主动创建数据容器并绑定监听器，不能依赖数据已存在**。面板创建（`createPanel`）和服务端数据到达（Handler 处理）的时序不确定，如果面板中以 `if (data != null)` 判断后才绑定监听器，当数据晚于面板到达时监听器不会被注册，导致后续数据变化无法刷新 UI。正确做法是在 `createPanel` 中始终通过 `getOrCreate` 模式获取数据容器（如 `ObservableList`），确保监听器一定被注册；Handler 中同样使用相同的 `getOrCreate` 模式，保证两端共享同一个容器实例

### 7.4 多账号隔离

- 每个账号的数据必须存储在 `ClientAccount.moduleDataMap` 中，禁止使用全局静态变量存储业务数据
- 面板每次通过 `createPanel(account)` 创建时绑定到指定账号，不共享面板实例
- 发送消息使用 `account.sendMessage(msg)` 而非直接操作 session

### 7.5 MCP 工具

- 新增业务模块的 MCP 工具方法统一写在 `mcp/ClientMcpTools.java` 中
- 方法使用 `@Tool` 注解声明，参数使用 `@ToolParam` 注解描述（`org.springframework.ai.tool.annotation`）
- 通过 `AccountManager` 获取账号上下文后操作
- 返回值使用 `String`，内容清晰易读（供 AI 解析）

## 8. 配置说明

```yaml
spring:
  application:
    name: slg-client
  ai:
    mcp:
      server:
        type: SYNC
        name: slg-client
        version: 1.0.0

server:
  port: 8099              # MCP Server 端口（SSE 传输复用 Tomcat）

client:
  server-url: ws://localhost:50001/ws   # 游戏服务器 WebSocket 地址
```

Cursor 通过 `.cursor/mcp.json` 配置连接：
```json
{
  "mcpServers": {
    "slg-client": {
      "url": "http://localhost:8099/sse"
    }
  }
}
```
